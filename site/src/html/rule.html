CRITICAL jwt-no-verification

JWT is decoded but never verified

CWE-347 OWASP A02:2021 ts · python · go

What it catches

Code that decodes a JSON Web Token to read its claims without ever verifying the signature. Decoding is a reversible base64 operation — it proves nothing about who issued the token. Acting on those claims lets any attacker forge a token and impersonate any user.

Why it matters

This is the single most common critical that AI assistants generate — jwt.decode() reads as harmless and the happy path works in tests. There is no error and no crash until an attacker sends {"sub":"admin"} with a throwaway signature.

VULNERABLE
auth/session.ts
const token = req.headers.authorization?.split(' ')[1]
const claims = jwt.decode(token)
req.userId = claims.sub // trusts an unverified token
SAFE
auth/session.ts
const claims = jwt.verify(token, publicKey, {
  algorithms: ['RS256'], // pin the algorithm
  issuer: 'https://auth.example.com',
  audience: 'api://default',
})
req.userId = claims.sub // signature, alg, iss & aud checked

Suppressing this rule

If a finding is a genuine false positive, scope the suppression to the exact line and leave a reason — never disable the rule project-wide.

// oauthlint-disable-next-line jwt-no-verification -- token pre-verified by gateway
const claims = jwt.decode(token)
