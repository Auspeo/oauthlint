Getting started

OAuthLint runs anywhere Node runs. There is nothing to install and no account to create — npx fetches the latest release and scans your source in one step.

Install & run

Point it at a directory. The first run downloads the rulepack and caches it.

terminal
$ npx oauthlint scan ./src
# scan a single file
$ npx oauthlint scan ./src/auth/session.ts

Add to CI

Drop the action into any workflow. Use fail-on to set the severity that blocks a merge, and sarif: true to surface findings in GitHub code scanning.

.github/workflows/auth.yml
- name: OAuthLint
  uses: oauthlint/scan-action@v1
  with:
    paths: ./src
    fail-on: high
    sarif: true

Reading the output

Every finding carries a severity, a rule id, the CWE it maps to, and the exact file:line:col. Because precision is the whole point, a critical means a real, reachable bypass — treat it as a blocker.

TIP
Run oauthlint scan --format json to pipe findings into your own tooling, or --format sarif for any SARIF-aware viewer.
